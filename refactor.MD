중요 요구사항:

1. 기존 기능과 UI 동작은 모두 그대로 유지해야 합니다.
2. 새로운 기능을 추가하는 작업이 아니라 구조 개선 작업입니다.
3. 함수 이름과 상태 흐름은 가능한 한 유지하되, 기능별 파일로 분리해 주세요.
4. 전역 상태가 여러 파일에 중복 생성되지 않도록 주의해 주세요.
5. 순환 참조가 생기지 않도록 의존성을 정리해 주세요.
6. 기존 Neutralino API, Settings, Queue, YTDlp 연동을 깨뜨리지 마세요.
7. 기존 플레이리스트 생성, 이름 변경, 삭제, 전환 기능을 모두 유지하세요.
8. 플레이리스트를 변경하면 현재 재생 중인 음악이 즉시 정지되고, 새 플레이리스트에서는 곡을 선택하기 전 상태가 유지되어야 합니다.
9. 플레이리스트를 삭제해도 실제 음악 파일과 전체 음악 목록은 삭제되면 안 됩니다.
10. 기존 저장 방식인 Settings.save()를 그대로 사용하세요.
11. localStorage 기반의 새로운 별도 저장 로직을 추가하지 마세요.
12. 리팩터링 후 모든 import 경로와 초기화 순서를 점검해 주세요.
13. 브라우저 콘솔에 오류가 발생하지 않도록 확인해 주세요.
14. HTML에서 script 태그 수정이 필요하다면 type="module" 적용까지 포함해 주세요.
